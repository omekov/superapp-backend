  -IPATH, --proto_path=PATH
   Укажите каталог, в котором следует искать 
импорт. Может быть указан несколько раз;
каталоги будут искаться по порядку. Если не
данный, используется текущий рабочий каталог.
Если он не найден ни в одном из этих каталогов,
дескрипторы --descriptor_set_in будут
проверено на наличие необходимого файла proto.

  --version 
Показать информацию о версии и выйти.

  -h, --help 
Показать этот текст и выйти.

  --encode=MESSAGE_TYPE 
  Прочитать текстовое сообщение заданного типа
из стандартного ввода и записать его в двоичном виде
на стандартный вывод. Тип сообщения должен
быть определены в PROTO_FILES или их импортах.

  --deterministic_output 
  При использовании --encode убедитесь, что поля карты
детерминистически упорядочены. Обратите внимание, что этот порядок
не является каноническим и меняется в разных сборках или
выпуски проток.

  --decode=MESSAGE_TYPE 
  Считать бинарное сообщение заданного типа из
стандартный ввод и записать его в текстовом формате
на стандартный вывод. Тип сообщения должен
быть определены в PROTO_FILES или их импортах.

  --decode_raw 
  Читать произвольное сообщение протокола из
стандартный ввод и запись необработанного тега/значения
пар в текстовом формате в стандартный вывод. Нет
PROTO_FILES следует указывать при использовании этого
флаг.

  --descriptor_set_in=PATH     
  Указывает разделенный список ФАЙЛОВ
каждый из которых содержит FileDescriptorSet (a
буфер протокола, определенный в descriptor.proto).
FileDescriptor для каждого из PROTO_FILES
при условии, что они будут загружены из этих
Наборы дескрипторов файлов. Если дескриптор файла
появляется несколько раз, первое появление
будет использован.

  -oFILE,                     
  Записывает FileDescriptorSet (буфер протокола, 
--descriptor_set_out=PATH, определенный в descriptor.proto), содержащий все
входные файлы в FILE.

  --include_imports         
  При использовании --descriptor_set_out также включать
все зависимости входных файлов в
набор, так что набор является самодостаточным.

  --include_source_info     
  При использовании --descriptor_set_out не удалять
SourceCodeInfo из файла FileDescriptorProto.
Это приводит к значительно большим дескрипторам, которые
включить информацию об оригинале
расположение каждого decl в исходном файле как
а также окружающие комментарии.

  --dependency_out=PATH     
  Записать выходной файл зависимостей в формате
ожидается по марке. Это пишет переходный
набор путей входных файлов к FILE

  --error_format=FORMAT     
  Установите формат, в котором будут печататься ошибки.
ФОРМАТ может быть «gcc» (по умолчанию) или «msvs».
(формат Microsoft Visual Studio).

  --fatal_warnings          
Сделать предупреждения фатальными (аналогично -Werr в
гкк). Этот флаг заставит protoc вернуться
с ненулевым кодом выхода, если какие-либо предупреждения
генерируются.

  --print_free_field_numbers 
 Распечатать номера свободных полей сообщений
определенные в данных прото-файлах. Группы делятся
тот же пробел номера поля с родителем
сообщение. Диапазоны расширений учитываются как
номера занятых полей.

  --plugin=EXECUTABLE       
  Указывает исполняемый модуль подключаемого модуля.
Обычно protoc ищет ПУТЬ для
плагины, но вы можете указать дополнительные
исполняемые файлы не в пути с использованием этого флага.
Кроме того, EXECUTABLE может иметь форму
NAME=PATH, в этом случае заданное имя плагина
сопоставляется с данным исполняемым файлом, даже если
собственное имя исполняемого файла отличается.

  --cpp_out=OUT_DIR Создать заголовок и исходный код C++.
  --csharp_out=OUT_DIR Создать исходный файл C#.
  --java_out=OUT_DIR Создать исходный файл Java.
  --js_out=OUT_DIR Создать исходный код JavaScript.
  --kotlin_out=OUT_DIR Создать файл Kotlin.
  --objc_out=OUT_DIR Создать заголовок и источник Objective-C.
  --php_out=OUT_DIR Создать исходный файл PHP.
  --python_out=OUT_DIR Создать исходный файл Python.
  --ruby_out=OUT_DIR Создать исходный файл Ruby.

  @<filename> 
  Чтение опций и имен файлов из файла. Если
указывается относительный путь к файлу, файл
будет искать в рабочем каталоге.
Параметр --proto_path не повлияет на то, как
этот файл аргументов ищется. Содержание
файл будет развернут в позиции
@<filename> как в списке аргументов. Примечание
что расширение оболочки не применяется к
содержимое файла (т. е. вы не можете использовать
кавычки, подстановочные знаки, escape-последовательности, команды и т.д.).
Каждая строка соответствует одному аргументу,
даже если он содержит пробелы.
